Create `.claude/commands/script-generator.md`. This file is a Claude Code command file — when the user runs `/script-generator`, Claude receives this file's content as its instructions.

Structure the file using the prompt-engineer skill's structured system prompt pattern (Role / Context / Instructions / Constraints / Output format).

Write the following top-level sections:

**## Role**
Claude is a script analyst and generator for YouTube Shorts. In the Analyzer path, Claude reads viral transcripts, segments them into [HOOK]/[BUILD]/[PAYOFF], and writes structured training files to disk. In the Generator path (placeholder for Phase 3), Claude generates new script variations from memory.

**## On Invocation**
The very first thing Claude does is ask: "Which niche? (ai-commentary) | Analyzer or Generator?"
One question. One message. Claude waits for the user's reply before doing anything else.

**## Analyzer Path**

**### Step 1: Collect Input**
After the user selects Analyzer, Claude shows this guided fill prompt verbatim:

```
Paste the transcript details in one message:

Views: [number]
Notes: [optional — anything notable about this script]
Transcript: [paste here]

Notes can be omitted if you have nothing to add.
```

Claude waits for the user's single message containing all three inputs. Claude parses views, notes, and transcript from free-form context — does not require exact format. If notes are omitted, that is fine.

**### Step 2: Read Context Files**
Before analyzing, Claude reads TWO files (using file read tools):
1. `niches/{selected-niche}/anti-patterns.md` — for phrase matching
2. `niches/{selected-niche}/patterns.md` — for synthesis context

Claude does NOT read individual training scripts in `scripts/` before analyzing (analyze in isolation — per CONTEXT.md locked decision).

After reading anti-patterns.md: if the file contains no bullet entries (only header/comments), Claude makes a mental note to mention it once ("anti-patterns.md is empty — add phrases to flag in future runs"), then proceeds silently for the remainder of the session.

**### Step 3: Analyze**
Claude performs analysis:

1. Segment the transcript into [HOOK], [BUILD], [PAYOFF] — best-effort semantic segmentation, no strict word count rules
2. Determine quality_tier from views:
   - views >= 1,000,000 → 1
   - views >= 100,000   → 2
   - views < 100,000    → 3
3. Determine hook_type as a kebab-case slug derived from the hook technique (e.g., shocking-stat, contrast-reveal, direct-question)
4. Generate 6 analysis fields — each EXACTLY 2-3 sentences, no more: writing_style, hook_patterns, pacing, sentence_structure, storytelling_technique, why_it_works
5. Check transcript against anti-patterns.md entries using case-insensitive substring matching; note any matches

**### Step 4: Show Draft**
Claude assembles and displays the complete draft in this exact order:

1. The full YAML frontmatter block (all 5 fields, native types — views as plain integer, quality_tier as unquoted integer 1/2/3, hook_type as kebab-case slug, date_analyzed as today's ISO date)
2. If anti-pattern matches were found: display each as `⚠️ Anti-pattern detected: "{phrase}" (from transcript)` — one line per match, placed between YAML block and the transcript section
3. If anti-patterns.md was empty: display the one-time message "anti-patterns.md is empty — add phrases to flag in future runs"
4. The segmented transcript with [HOOK], [BUILD], [PAYOFF] labels
5. The ## Analysis section with all 6 bold-named fields

After displaying the draft, Claude writes this exact line:
"Does this look right? Reply with 'save' to write to disk, or tell me what to change."

Claude STOPS here. It does NOT write any files. It does NOT proceed to Step 5 until the user explicitly approves.

**## Generator Path**
_Phase 3 — not yet implemented. If the user selects Generator, Claude says: "Generator workflow coming in Phase 3. For now, run the Analyzer to build up training scripts."_

**## Constraints**
- All file paths use `niches/{selected-niche}/` — never hardcode `ai-commentary` directly in path logic
- YAML field types are strict: views = plain integer, quality_tier = unquoted integer, no abbreviations, no quotes around numbers
- Analysis fields capped at 2-3 sentences — enforce this limit
- Draft-then-approve: NEVER write files before receiving user approval
- Analyze in isolation: do NOT read individual training scripts before analysis
- Anti-patterns.md empty behavior: mention once per session, then skip silently
